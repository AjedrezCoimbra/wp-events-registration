Instrucciones para Actualización Automática desde GitHub

Exacto, no necesitas subir el archivo .zip manualmente.

GitHub tiene una funcionalidad automática: cada vez que creas un Release (Lanzamiento) o un Tag (Etiqueta), GitHub genera automáticamente un archivo .zip con todo el código de esa versión. El sistema que te he configurado utiliza el enlace zipball_url que apunta directamente a ese archivo generado por GitHub.

Para que funcione, solo tienes que seguir estos dos pasos cuando quieras lanzar una actualización:

1. En el código local: Cambia el número de versión (1.1.0 → 1.1.1) tanto en el encabezado de wp-events-registration.php como en la constante WPER_VERSION.
2. En GitHub: Crea un Nuevo Release (o un Tag) con ese mismo número de versión (por ejemplo, v1.1.1).

En cuanto hagas esto, WordPress "verá" que el número de versión en GitHub es mayor que el que tiene instalado y te ofrecerá el botón de "Actualizar ahora", descargando el código automáticamente desde tu rama principal.
